# シミュレーション設計

## 目的

このシミュレーションは、創造性全体を証明するものではありません。

目的は、次の限定仮説を人工ワールドモデル上で検査することです。

> 内部不整合を持つモデルに対して、再解釈探索はランダム修復よりも高い評価スコアを持つ代替モデルを生成できる場合がある。

## 比較する方法

### 1. ランダム修復

内部不整合をランダムに解消します。

これは偶然との差を見るためのベースラインです。

### 2. 教師モデル復元

教師モデルまたは参照モデルへ戻します。

これは「正解復元」との比較です。創造性候補としては弱いですが、整合性は高くなります。

### 3. 再解釈探索

次の条件を同時に満たす候補を探索します。

- 内部的に整合している。
- 初期モデルの一部を保存している。
- 元モデルと異なる。
- 有用性 proxy が高い。

## 評価指標

| 指標 | 意味 |
|---|---|
| 整合性 | 肯定・否定の衝突がないか |
| 新規性 | 教師モデルまたは元モデルと異なるか |
| 保存度 | 初期モデルの情報をどれだけ残したか |
| 有用性 proxy | 構造が空でなく、一定の関係を持つか |
| 再解釈スコア | 上記を合成した値 |

## 固定デモ

`scripts/simulate.py` は、3ノードの固定ケースを実行します。

```bash
python scripts/simulate.py
```

このデモは、モデルの意味を確認するための最小例です。

## 複数ケース実験

`scripts/run_experiments.py` は、複数 seed の人工ケースを実行します。

```bash
python scripts/run_experiments.py
```

現時点では、次の条件です。

| 項目 | 値 |
|---|---:|
| ノード数 | 3 |
| 試行数 | 30 seed |
| conflict_rate | 0.35 |
| extra_positive_rate | 0.25 |

初期結果は [`../results/initial_batch_summary.md`](../results/initial_batch_summary.md) に保存しています。

## 合否条件

PASS条件は次です。

```text
平均スコア(再解釈探索) > 平均スコア(ランダム修復)
```

かつ、再解釈探索が単に教師モデルへ戻るだけではないこと。

FAIL条件は次です。

```text
平均スコア(再解釈探索) <= 平均スコア(ランダム修復)
```

または、再解釈結果がほぼ教師モデル復元だけになること。

## 注意

このシミュレーションの価値関数は proxy です。

つまり、ここでの「価値」は、人間の価値判断を直接表すものではありません。最初の段階では、モデルが検証可能な形で動くかを確認するための簡易指標です。

また、現時点の再解釈探索は全候補探索に近いため、ランダム修復より有利です。今後は、探索コストや候補数制約を入れて、より厳しい比較にする必要があります。

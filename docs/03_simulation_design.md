# シミュレーション設計

## 目的

このシミュレーションは、創造性全体を証明するものではありません。

目的は、次の限定仮説を人工ワールドモデル上で検査することです。

> 内部不整合を持つモデルに対して、再解釈探索は単発ランダム修復よりも高い構造 proxy スコアを持つ代替モデルを生成できる場合がある。

ただし、単発ランダム修復だけではベースラインが弱いので、同じ候補数予算を使うランダム探索、局所修復探索も比較対象に入れます。

## 比較する方法

### 1. 単発ランダム修復

内部不整合をランダムに1回だけ解消します。

これは最も弱いベースラインです。

### 2. ランダム探索

同じ候補数予算を使って、複数の候補モデルをランダムに生成し、その中で最も高いスコアを採用します。

これは単発ランダム修復より強いベースラインです。

### 3. 局所修復探索

初期モデルから始め、1辺だけ変更した近傍候補を評価し、スコアが改善する候補へ移動します。

改善が止まった場合は、1辺だけ摂動して探索を続けます。

これは、初期理解を完全に捨てるのではなく、現在の理解を少しずつ修正する探索です。

### 4. 教師モデル復元

教師モデルまたは参照モデルへ戻します。

これは「正解復元」との比較です。創造性候補としては弱いですが、整合性は高くなります。

### 5. 再解釈探索

次の条件を同時に満たす候補を探索します。

- 内部的に整合している。
- 初期モデルの一部を保存している。
- 元モデルと異なる。
- utility_proxy が高い。

## 評価指標

| 指標 | 意味 |
|---|---|
| 整合性 | 肯定・否定の衝突がないか |
| 新規性 | 教師モデルまたは元モデルと異なるか |
| 保存度 | 初期モデルの情報をどれだけ残したか |
| utility_proxy | 密度・ノード被覆・弱連結・入出力被覆の合成値 |
| 再解釈スコア | 新規性・保存度・utility_proxy の積 |

## utility_proxy の内訳

現在の `utility_proxy` は、次の4要素からなる暫定指標です。

| 構成要素 | 意味 |
|---|---|
| density_score | 肯定辺の本数がノード数付近なら高い。空構造と過密構造を下げる。 |
| node_coverage_score | 肯定辺がどれだけ多くのノードを覆っているか。 |
| weak_connectivity_score | 肯定辺を無向辺として見たとき、最大連結成分がどれだけ大きいか。 |
| in_out_coverage_score | 肯定辺について、入辺・出辺を持つノードがどれだけ広がっているか。 |

合成値は次です。

```text
utility =
  0.35 * density_score
  + 0.25 * node_coverage_score
  + 0.25 * weak_connectivity_score
  + 0.15 * in_out_coverage_score
```

この重みは暫定値です。人間の価値判断ではありません。

詳細は [`10_utility_proxy.md`](10_utility_proxy.md) を参照してください。

## 固定デモ

`scripts/simulate.py` は、3ノードの固定ケースを実行します。

```bash
python scripts/simulate.py
```

このデモは、モデルの意味を確認するための最小例です。

## 複数ケース実験

`scripts/run_experiments.py` は、複数 seed / 複数ノード数の人工ケースを実行します。

```bash
python scripts/run_experiments.py \
  --nodes 3,4,5 \
  --trials 10 \
  --candidate-limit 200
```

現時点では、次の条件です。

| 項目 | 値 |
|---|---:|
| ノード数 | 3, 4, 5 |
| 試行数 | 各10 seed |
| candidate_limit | 200 |
| conflict_rate | 0.35 |
| extra_positive_rate | 0.25 |

初期結果は [`../results/initial_batch_summary.md`](../results/initial_batch_summary.md) に保存しています。

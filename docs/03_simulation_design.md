# シミュレーション設計

## 目的

このシミュレーションは、創造性全体を証明するものではありません。

目的は、次の限定仮説を人工ワールドモデル上で検査することです。

> 内部不整合を持つモデルに対して、再解釈探索はランダム修復よりも高い評価スコアを持つ代替モデルを生成できる場合がある。

ただし、単発ランダム修復だけではベースラインが弱いので、同じ候補数予算を使うランダム探索も比較対象に入れます。

## 比較する方法

### 1. 単発ランダム修復

内部不整合をランダムに1回だけ解消します。

これは最も弱いベースラインです。

### 2. ランダム探索

同じ候補数予算を使って、複数の候補モデルをランダムに生成し、その中で最も高いスコアを採用します。

これは単発ランダム修復より強いベースラインです。

### 3. 教師モデル復元

教師モデルまたは参照モデルへ戻します。

これは「正解復元」との比較です。創造性候補としては弱いですが、整合性は高くなります。

### 4. 再解釈探索

次の条件を同時に満たす候補を探索します。

- 内部的に整合している。
- 初期モデルの一部を保存している。
- 元モデルと異なる。
- 有用性 proxy が高い。

## 評価指標

| 指標 | 意味 |
|---|---|
| 整合性 | 肯定・否定の衝突がないか |
| 新規性 | 教師モデルまたは元モデルと異なるか |
| 保存度 | 初期モデルの情報をどれだけ残したか |
| 有用性 proxy | 構造が空でなく、一定の関係を持つか |
| 再解釈スコア | 上記を合成した値 |

## 固定デモ

`scripts/simulate.py` は、3ノードの固定ケースを実行します。

```bash
python scripts/simulate.py
```

このデモは、モデルの意味を確認するための最小例です。

## 複数ケース実験

`scripts/run_experiments.py` は、複数 seed / 複数ノード数の人工ケースを実行します。

```bash
python scripts/run_experiments.py \
  --nodes 3,4,5 \
  --trials 10 \
  --candidate-limit 200
```

現時点では、次の条件です。

| 項目 | 値 |
|---|---:|
| ノード数 | 3, 4, 5 |
| 試行数 | 各10 seed |
| candidate_limit | 200 |
| conflict_rate | 0.35 |
| extra_positive_rate | 0.25 |

初期結果は [`../results/initial_batch_summary.md`](../results/initial_batch_summary.md) に保存しています。

## 合否条件

弱いPASS条件:

```text
平均スコア(再解釈探索) > 平均スコア(単発ランダム修復)
```

強いPASS条件:

```text
平均スコア(再解釈探索) > 平均スコア(ランダム探索)
```

現時点では、弱いPASS条件は満たしますが、強いPASS条件はノード数4〜5で満たしていません。

## 注意

このシミュレーションの価値関数は proxy です。

つまり、ここでの「価値」は、人間の価値判断を直接表すものではありません。最初の段階では、モデルが検証可能な形で動くかを確認するための簡易指標です。

また、現時点の再解釈探索は、強めのランダム探索に負ける条件があります。したがって、今後は探索方針・価値関数・保存度の定義を改善する必要があります。

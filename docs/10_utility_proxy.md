# utility_proxy 設計メモ

この文書では、シミュレーションで使う `utility_proxy` の意味と限界を整理します。

## 1. 前提

`utility_proxy` は、人間の価値判断ではありません。

これは、人工ワールドモデル上で候補構造を比較するための簡易代理指標です。

したがって、`utility_proxy` が高いことは、その候補が本当に創造的であることを意味しません。

## 2. なぜ改善したか

以前の `utility_proxy` は、ほぼ肯定辺の本数だけで決まっていました。

これは単純すぎます。

問題は次です。

- 辺の本数だけでは、構造が全体を覆っているか分からない。
- 一部ノードだけに集中した構造を高く評価する可能性がある。
- ばらばらな関係集合でも、辺の本数だけで高くなる可能性がある。
- 入力・出力の向きが偏っていても評価できない。

そのため、現在は次の4つに分解しています。

## 3. 構成要素

| 構成要素 | 意味 |
|---|---|
| density_score | 肯定辺の本数がノード数付近なら高い。空構造と過密構造を下げる。 |
| node_coverage_score | 肯定辺がどれだけ多くのノードを覆っているか。 |
| weak_connectivity_score | 肯定辺を無向辺として見たとき、最大連結成分がどれだけ大きいか。 |
| in_out_coverage_score | 肯定辺について、入辺・出辺を持つノードがどれだけ広がっているか。 |

合成値は次の重み付き平均です。

```text
utility =
  0.35 * density_score
  + 0.25 * node_coverage_score
  + 0.25 * weak_connectivity_score
  + 0.15 * in_out_coverage_score
```

## 4. 意図

この proxy は、次のような候補を相対的に高く評価します。

- 空ではない。
- 過密すぎない。
- 複数ノードを覆う。
- ばらばらではなく、ある程度つながっている。
- 入出力の向きが一部ノードに偏りすぎない。

つまり、単なる差分やランダムな辺集合ではなく、最低限の構造を持つ候補を評価するための指標です。

## 5. 限界

この指標には明確な限界があります。

- 人間の有用性を表していない。
- 芸術的価値を表していない。
- 科学的真理性を表していない。
- 技術的実装可能性を表していない。
- 重みは暫定値である。
- 特定のグラフ形状を優遇する可能性がある。

したがって、この指標は人間の価値そのものではなく、**構造的な proxy** です。

## 6. 今後の改善案

今後は、次を検討できます。

- 重みの感度分析
- 複数 utility proxy の比較
- 説明圧縮率との組み合わせ
- 課題達成率との組み合わせ
- 教師モデルとの差分と保存度のトレードオフ分析

ただし、これらを入れすぎるとリポジトリの範囲が広がります。

現段階では、`utility_proxy` を分解可能にし、限界を明記するところまでに留めます。

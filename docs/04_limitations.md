# 限界

このリポジトリでは、意図的に射程を狭くしています。

## 1. 創造性全体を説明しない

このモデルは、創造性の一部である **再解釈型創造性** だけを扱います。

創造性には、熟練者の設計、ランダム探索、偶然の発見、組合せ探索、美的判断、社会的流行など、多数の経路があります。それら全体をこのモデルで説明することはできません。

## 2. 人間心理を直接観測していない

このモデルのワールドモデルは、人工的なグラフです。

人間の内部ワールドモデルを直接観測しているわけではありません。そのため、人間心理の実証モデルではなく、仮説を整理するための計算モデルです。

## 3. 自然言語を直接扱わない

同じ文章でも、主体のワールドモデルが違えば解釈は変わります。

しかし、初期実装では自然言語処理を扱いません。自然言語を扱うと、意味論・文脈・語用論まで範囲が広がりすぎるためです。

## 4. 内部不整合の定義が単純

初期実装では、内部不整合を「同じ関係の肯定と否定が同時に存在すること」として扱います。

これは非常に単純な定義です。現実には、意味矛盾、行動矛盾、因果方向の衝突、価値判断の衝突などがあります。

## 5. 価値関数は代理指標である

シミュレーション中の有用性・価値は proxy です。

人間の審美性、科学的価値、技術的有用性、社会的受容を直接測るものではありません。

## 6. シミュレーションは実証ではない

シミュレーションは、仮説を検査する補助です。

シミュレーションで良い結果が出ても、人間の創造性が証明されたことにはなりません。

## 7. 形式化は創造性の証明ではない

将来的に Lean などで形式化しても、証明できるのは定義間の関係です。

Lean によって「創造性そのもの」が証明されるわけではありません。

## まとめ

このリポジトリの位置づけは、完成理論ではなく、**再解釈型創造性を検証可能な仮説として整理するための研究メモ兼最小実装**です。

# 探索方針メモ

この文書では、再解釈探索が強めのランダム探索や局所修復探索に負ける理由と、今後の探索方針を整理します。

## 1. 現状の問題

現在の再解釈探索は、単発ランダム修復には勝ちます。

一方で、同じ候補数予算を使うランダム探索には、ノード数4〜5で負ける条件があります。

さらに、局所修復探索はノード数4〜5で現在の再解釈探索より高い代理スコアを出しています。

これは、現在の再解釈探索がまだ十分に強い探索戦略ではないことを示します。

## 2. なぜ負けるのか

主な原因は次です。

| 原因 | 内容 |
|---|---|
| 初期モデル保存バイアス | 初期モデルを保存しすぎると、誤った前提も保存される |
| 候補生成の狭さ | 再解釈候補が初期モデル周辺に偏る |
| ランダム探索の強さ | 候補数予算が同じなら、広い探索を行うランダム探索が高得点候補を拾う場合がある |
| 局所改善の不足 | 現在の再解釈探索は、1辺ずつ改善する探索過程を明示的に持たない |
| utility_proxy 依存 | 評価関数が特定の構造を有利にする場合がある |

## 3. 局所修復探索

局所修復探索は、完全なランダム探索ではなく、初期モデルから小さな変形を繰り返してスコアを改善します。

```text
初期モデル
  -> 1辺を変更した近傍候補を作る
  -> スコアが上がる候補へ移動する
  -> 改善が止まったら1辺だけ摂動する
  -> 予算内で繰り返す
```

これは、人間の再解釈における「全く別の構造をランダムに探す」のではなく、「現在の理解を少しずつ修正する」に近い探索です。

## 4. 比較対象

今後の比較は、少なくとも次を分けます。

| 方法 | 意味 |
|---|---|
| 単発ランダム修復 | 矛盾を1回だけランダム解消する弱いベースライン |
| ランダム探索 | 同じ候補数予算で広く候補を探す強いベースライン |
| 局所修復探索 | 初期モデル近傍を少しずつ改善する探索 |
| 再解釈探索 | 現在の再解釈候補生成方式 |

## 5. 現時点の結果

初期条件では、次の傾向があります。

- 再解釈探索は単発ランダム修復より高い。
- 再解釈探索は強めのランダム探索に負ける条件がある。
- 局所修復探索はノード数4〜5で現在の再解釈探索より高い。

この結果は、局所修復探索が「真の創造性」に近いことを意味しません。

意味するのは、現在のスコア関数と候補生成条件では、局所改善を明示的に入れた探索の方が有利な場合がある、ということです。

## 6. 注意

局所修復探索を追加しても、それが創造性を証明するわけではありません。

局所修復探索は、再解釈型創造性の候補生成をより説明しやすくするための実装上の改善です。

## 7. 次の改善案

次に検討すべきなのは、次です。

1. 再解釈探索と局所修復探索の統合
2. 初期モデル保存度を調整する温度パラメータ
3. 探索コストを明示した公平比較
4. 候補がどの辺を変更したかの説明ログ
5. スコア改善過程の保存

## 8. 現時点の判断

局所修復探索は、このリポジトリの範囲内で追加してよい最小改善です。

一方、生徒集団・アンサンブル・大規模ネットワークは、まだ追加しません。

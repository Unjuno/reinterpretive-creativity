# モデル定義

## 基本概念

このモデルでは、主体が持つ認識構造を **ワールドモデル** と呼びます。

ワールドモデルは、概念ノードと関係エッジからなる単純な構造として扱います。

- ノード: `A`, `B`, `C` などの概念
- 肯定関係: `A -> B`
- 否定関係: `not (A -> B)`

初期実装では、自然言語そのものは扱いません。自然言語を直接扱うと範囲が広がりすぎるためです。

## 記号

| 記号 | 意味 |
|---|---|
| `W` | 初期ワールドモデル |
| `W'` | 再解釈後ワールドモデル |
| `Inc(W)` | `W` に内部不整合がある |
| `Con(W')` | `W'` が内部的に整合している |
| `Diff(W, W')` | `W` と `W'` が異なる |
| `Eval_proxy(W')` | `W'` が明示された構造的 proxy で評価される |
| `RIC_candidate(W, W')` | 再解釈型創造性候補 |

## 内部不整合

初期実装では、内部不整合を次のように限定します。

```text
A -> B
not (A -> B)
```

同じ関係を肯定し、同時に否定している場合、内部不整合があるとみなします。

これは単純化です。現実の人間の認識では、意味矛盾、行動矛盾、因果方向の衝突などもあります。しかし、MVPでは扱いません。

## 再解釈

再解釈とは、内部不整合を持つ初期モデル `W` から、内部的に整合した代替モデル `W'` を作ることです。

ただし、教師モデルや元モデルへ完全に戻るだけでは、ここでいう再解釈型創造性候補としては弱いです。

再解釈後モデルには、少なくとも次が必要です。

1. 不整合が解消されている。
2. 元モデルと異なる。
3. 明示された構造的 proxy によって評価される。

## 定義

```text
RIC_candidate(W, W') :=
  Inc(W)
  and Con(W')
  and Diff(W, W')
  and Eval_proxy(W')
```

この定義は、創造性全体の定義ではありません。

これは、再解釈型創造性候補の最小定義です。

## 重要な区別

| 区別 | 内容 |
|---|---|
| 誤り | 内部不整合や誤認識そのもの |
| 修復 | 不整合を減らす操作 |
| 再解釈 | 修復後に元と異なる整合的構造が生まれること |
| 創造性候補 | 再解釈後構造が明示された構造的 proxy によって評価される場合 |
| 社会的創造性 | 他者・共同体・専門家などが価値を認める場合 |

このリポジトリでは、主に「構造的 proxy によって評価される創造性候補」までを扱います。

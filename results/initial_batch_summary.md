# 初期バッチ実験サマリ

この結果は、`scripts/run_experiments.py` の固定設定で得られる決定的な初期デモです。

現在の位置づけでは、このファイルは historical initial demo です。Phase 1 current-scope closure の根拠全体ではなく、初期実装が動作したことを示す補助的な結果として扱います。

## 条件

| 項目 | 値 |
|---|---:|
| ノード数 | 3, 4, 5 |
| 試行数 | 各10 seed |
| candidate_limit | 200 |
| conflict_rate | 0.35 |
| extra_positive_rate | 0.25 |
| 比較1 | 単発ランダム修復 vs 再解釈探索 |
| 比較2 | 同じ候補数予算のランダム探索 vs 再解釈探索 |
| 比較3 | 局所修復探索 vs 再解釈探索 |
| utility_proxy | 密度・ノード被覆・弱連結・入出力被覆の合成 |

実行コマンド:

```bash
python scripts/run_experiments.py \
  --nodes 3,4,5 \
  --trials 10 \
  --candidate-limit 200
```

## 解釈

この条件では、再解釈探索は単発ランダム修復より高い proxy score を出しました。

一方、同じ候補数予算を使う強めのランダム探索と比べると、ノード数4〜5では再解釈探索が負けています。

さらに、局所修復探索はノード数4〜5で再解釈探索より高い proxy score を出しました。これは、初期モデル近傍を1辺ずつ改善する探索の方が、現時点の再解釈候補サンプリングより有利な条件があることを示します。

## 制約

この結果は、創造性全体を証明するものではありません。次の制約があります。

- 小規模人工モデルである。
- `utility_proxy` は構造的 proxy であり、人間の価値判断ではない。
- 人間評価ではない。
- ノード数4以上では全候補探索ではなく候補サンプリングを使っている。
- 強めのランダム探索に負ける条件がある。
- 局所修復探索の改善は、現行スコア関数と候補生成条件に依存している。
- utility_proxy の重みは暫定値である。

したがって、この結果は「再解釈型創造性モデルの初期デモが動作している」ことは示しますが、「再解釈探索が十分に強い」とは示しません。

Phase 1 current-scope closure の現在の整理は、`docs/phase1_closure_summary.md` と `docs/closure_criteria.md` を参照してください。
